# A*STAR DAC Analysis

This repository contains the measurement data, analysis scripts, plots, and generated lookup tables from a 16-bit DAC linearity study. The work focuses on characterising the measured DAC transfer curve, identifying non-monotonic or discontinuous behaviour, and evaluating correction approaches using lookup-table calibration and oversampled/PWM-style interpolation.

The source measurement file is `DAC_data.csv`, which maps each digital code to the measured analog output voltage. The scripts in this repository use that sweep to calculate differential non-linearity (DNL), integral non-linearity (INL), residual error after correction, inferred segmentation behaviour, and oversampling-based output refinement.

## Project Goals

- Plot and inspect the raw DAC transfer curve across the full digital code range.
- Detect code regions where the measured output drops or deviates from the expected monotonic response.
- Estimate DNL and INL from measured voltage steps.
- Build lookup-table based corrections that map ideal target outputs to the closest measured DAC outputs.
- Export corrected lookup data for later firmware, simulation, or calibration use.
- Investigate DAC segmentation behaviour by locating recurring DNL spike intervals.
- Test oversampling and PWM-style interpolation as a way to approximate voltages between available DAC codes.

## Analysis Workflow

1. Start with the measured transfer curve in `DAC_data.csv`.
2. Plot the raw code-to-voltage relationship using `raw_data_plot.py`.
3. Identify non-monotonic drops in the measured output and interpolate corrected values with `edit_plot.py`.
4. Calculate DNL and INL using `edit_plot_inl+dnl_cal.py`.
5. Generate a lookup-table correction using `lookuptable_INL+DNL.py`.
6. Review exported correction data in `lookup.csv` and the `lookup_*.npy` files.
7. Quantify post-correction residual error using `residual_analysis.py`.
8. Investigate segmented DAC behaviour with `histogramanalysisforsegmentation_.py` and `thermometer_sweep.py`.
9. Explore fractional-code output approximation using `oversampled_pulse.py` and `oversamplingINL/oversamplingINL/DNL.py`.

## Method Summary

The DAC output was compared against an ideal straight-line transfer function. Voltage step size was converted into LSB units so local error could be expressed as DNL. Cumulative DNL was then used to estimate INL. For correction, the measured output values were searched against ideal target values so each ideal code could be mapped to the closest available measured DAC output. The resulting correction table was exported both as CSV and NumPy arrays.

The segmentation analysis looks for recurring gaps between large DNL spikes. This helps infer whether the DAC behaviour is related to internal coarse/fine or thermometer/binary code boundaries. The oversampling scripts then explore whether rapidly alternating between nearby DAC codes can approximate intermediate voltages after low-pass filtering.

## Running the Scripts

Install the Python dependencies:

```bash
pip install -r requirements.txt
```

Run scripts from the repository root so they can find `DAC_data.csv`:

```bash
python raw_data_plot.py
python lookuptable_INL+DNL.py
python residual_analysis.py
python thermometer_sweep.py
```

Most scripts display plots using Matplotlib. Some scripts also write generated outputs such as `lookup.csv`, `lookup_dnl.npy`, `lookup_inl.npy`, `pwm.csv`, `pwm_dnl.npy`, and `pwm_inl.npy`.

## Key Outputs

- `lookup.csv`: corrected DNL and INL values for the measured DAC code range.
- `lookup_*.npy`: binary NumPy exports for downstream analysis or simulation.
- `pwm.csv`: PWM/oversampling DNL and INL results across fractional input codes.
- Plot images: quick visual references for the measured and corrected DAC behaviour.

## Notes

- `lookup_tablecode(wrong ideology).py` is intentionally retained as a record of an earlier modelling attempt. The main lookup correction path is `lookuptable_INL+DNL.py`.
- The scripts assume the input CSV uses the column names `Decimal Value` and `Analog Output (V)`.
- The repository name uses `A-Star_DAC_Analysis` because GitHub repository names cannot contain `*`.
